# Engineering Guidance and Standards
 
This is the home of engineering guidance and standards for the Home Office. Learn more about this project on the [about page](https://engineering.homeoffice.gov.uk/about/).

It is built using Markdown, GOV.UK templates, HO styles, the [x-gov Eleventy Plugin](https://x-govuk.github.io/govuk-eleventy-plugin/) and GitHub Actions.

## Requirements

- [Node.js](https://nodejs.org)
- [Eleventy](https://www.11ty.dev)
- [x-gov Eleventy Plugin](https://x-govuk.github.io/govuk-eleventy-plugin/)
- [Nunjucks](https://mozilla.github.io/nunjucks/) for the templating language

## Installation

```
npm install
```

This will install the dependencies needed to run the site.

## Preview your changes locally

Eleventy has the ability to serve the site in a hot reload environment.  You can call this from the npm scripts:

```
npm run serve
```

This maps to Eleventy hot reload script 'serve'.

### Alternative method for previewing changes locally

Alternatively, to run the site locally you can build the static html files and then deploy the _site folder to a http server.

Firstly you can run

```
npm run build
```

Which creates the static html. Then you can use a good very light weight http server to host them.

```
npm install -g http-server
```

Installs the server globally and then navigate to a terminal/cmd prompt to the _site directory and run: 

```
http-server -p 8080
```

Now you can preview the site on http://localhost:8080

## Redirects and deprecated content

This site supports redirects for deprecated guidance pages using static redirect pages generated by Eleventy.

Redirect mappings are defined in [./_data/redirects.json](./_data/redirects.json).

Each redirect entry supports:

- `from`: the legacy path to redirect from (must be absolute and end with `/`)
- `to`: the replacement path to redirect to (must be absolute and end with `/`)
- `statusCode`: metadata only (for future platform-level redirects)
- `reason`: short explanation of why the redirect exists
- `replacementTitle`: optional human-readable target title shown on fallback content

Example:

```json
[
	{
		"from": "/principles/write-maintainable-reusable-and-evolutionary-code/",
		"to": "/principles/continuously-maintain-and-evolve/",
		"statusCode": 301,
		"reason": "Superseded by more comprehensive guidance",
		"replacementTitle": "Continuously maintain and evolve"
	}
]
```

When deprecating a content page, add metadata to the page front matter:

- `status: deprecated`
- `deprecatedOn: YYYY-MM-DD`
- `deprecationReason: <summary>`
- `replacedBy: /new-path/`

If you need the legacy URL to become a redirect, move the original content to a new permalink (for example an `-archived` suffix) so the redirect page can be generated at the original `from` path.

### Redirect limitations

These are static redirect pages that use canonical, meta refresh, and JavaScript navigation for compatibility. They are not native HTTP `301`/`308` responses at the hosting layer.
